# Mongoyia 优化整改方案 - 2026-07-01

## 目标

把当前系统从“Phase 10-15 代码和公开入口基本可用”推进到“全角色右侧浏览器验收可闭环、移动/小程序可用、生产 GO/NO-GO 证据完整”的状态。

## P0：必须优先修复

### 1. 小程序 `URLSearchParams` 兼容错误

- 现象：小程序页面弹出 `Can't find variable: URLSearchParams`。
- 定位：`frontend/modules/mall/views/chat/index.php` 存在 `new URLSearchParams(...)`。
- 影响：小程序/低版本 WebView 中客服聊天脚本可能直接中断，影响“联系客服”和“我的”页体验。
- 整改：
  - 替换为兼容函数，例如手动 `encodeURIComponent` 拼接 query。
  - 同步排查 `new URL(...)`、`FormData`、`Blob`、`MediaRecorder`、`crypto.randomUUID` 等 WebView 兼容对象。
  - 增加静态检查：小程序/H5 入口禁止直接使用未 polyfill 的现代浏览器 API。
- 验收：
  - 微信小程序环境打开“我的”、客服聊天、商品详情、订单页无弹窗报错。
  - PC/H5 客服聊天仍可发送文本、图片/文件/视频/语音入口仍按权限展示。

### 2. 恢复右侧浏览器全角色可控验收

- 现象：本轮没有暴露可调用的 in-app browser 自动化接口，无法由工具直接点击右侧浏览器。
- 影响：后台 admin、seller、买家登录态、客服闭环、分销员闭环不能在本轮自动验收中判定通过。
- 整改：
  - 恢复 Codex in-app browser 控制通道，或安排人工按文档矩阵在右侧浏览器逐项复核。
  - 复测前确保宝塔已拉取 `68cd50c` 或更新提交，并清 PHP/opcache/页面缓存。
- 验收矩阵：
  - 平台管理员：运营配置、支付统计、通知、客服、商品、物流、评论、分销页面。
  - 商家：`zhishichanquan / 123456` 登录，商品、订单、发货、物流、优惠券、统计、客服隔离。
  - 买家：注册/登录、商品、购物车、结算、订单、收藏、评论、客服。
  - 客服：买家发消息、后台接收、客服回复、工单/投诉/满意度。
  - 分销员：教程、素材、推广链接、业绩、提现申请入口。

### 3. 排查测试站 `HTTP 444`

- 现象：
  - 本地脚本访问 `/backend/*` 返回 `444`。
  - 登录/注册 POST 返回 `444`。
- 影响：HTTP 自动化无法覆盖后台和登录后表单流程；也可能影响外部监控/健康检查。
- 整改：
  - 在宝塔/Nginx/WAF/安全插件中查 444 规则命中原因。
  - 区分“拦截非浏览器脚本”与“应用真实错误”。
  - 对验收机 IP 或验收路径配置安全白名单，保留生产防护。
  - 如果是应用层返回，补日志和友好错误页，避免空响应。
- 验收：
  - 右侧浏览器仍正常。
  - 验收脚本至少可读取后台登录页、提交测试登录、打开只读后台矩阵。
  - 生产正式环境不放宽核心安全策略。

### 4. 后台注销/商家切换复测

- 现状：本地提交 `68cd50c` 已新增 `/backend/site/switch-login`，用 CSRF POST 桥接退出。
- 整改：
  - 宝塔拉取最新代码。
  - 清 opcache。
  - 右侧浏览器点击后台“注销”。
  - 确认跳到 `/backend/site/login`。
  - 登录 `zhishichanquan / 123456` 做商家角色验收。
- 验收：
  - 不再出现 tokenized logout URL。
  - 不再停留在 admin 会话。
  - Seller 菜单和数据隔离符合预期。
